Wait for Event: Effective To

You can specify effective to date for the association in this property window. If left blank, will be valid always. To open this window, click the edit button for the 'Effective To' property.

Functionality

The activity will wait for events starting from the Date and Time specified in the Effective From field till the Date and Time specified in this field.

  • If only date is specified in the Effective To field and time is not specified, then the activity will wait till the end of the day specified in Effective To field.
  • If both date and time is not specified in the Effective To field, then the activity will continue to wait for an event always, starting from the the Date and Time specified in the Effective From field.

Illustration 1

  • Effective From field is 08/10/2010 00:00:00
  • Effective To field is  08/18/2010 22:00:00

The activity will wait for events starting at 00:00:00 hours on 10th August 2010 till 22 hours (10:00 pm) on 18th August 2010.

Illustration 2

  • Effective From field is 08/10/2010 00:00:00
  • Effective To field is  08/18/2010

The activity will wait for events starting at 00:00:00 hours on 10th August 2010 till 23:59 hours (11:59 pm) on 18th August 2010.

Illustration 3

  • Effective From field is 08/10/2010 00:00:00
  • Effective To field is  blank

The activity will continue to wait for events starting at 00:00:00 hours on 10th August 2010 till the workflow is aborted or activity is paused or killed.

Note: You can also specify the time in the text editor. The time entered can be in either 24 hour or 12 hour format. If time is entered in 12 hour format then AM/PM has to be specified.

Example:

Date and time in 24 hour format

08/18/2010 22:30:00

Date and time in 12 hour format

08/18/2010 10:30:00 PM

The window has a text editor to enter the date. You can do this either by entering a discrete value directly in the text editor or by using the Field controls at the top of the window. These controls are described below:

  • The value type drop down - You can select the type of value here. You can specify SendTo, Variable, Content, or XMLVariables here.
  • The Value text box/drop down - You can specify the actual value of the Redirect URL. The control displayed depends on the option selected in the first drop down. If you had selected "Value", it will display a text box here. You can enter the Custom Document View URL in this box. If you had selected "Variable", "Content", or "XMLVariables" instead, there will be a drop down instead of the field. This drop down will have the variable collection or content collection or XMLVariables collection from which you can select an appropriate option to assign the Custom Document View URL.
  • Append - You can click this button to add the specified value, variable or content to the text editor.

The text tool bar above the text editor allows you to manage and format. It has the following text tools –

  • Cut (Ctrl+X) - to cut text
  • Copy (Ctrl+C) - to copy text
  • Paste (Ctrl+V) - to paste text
  • Undo (Ctrl+Z) - to undo the last action
  • Redo (Ctrl+Y) - to undo the last action
  • Repeat Last Command (F4) - to repeat the last command that was executed

It also has the following formatting & other tools –

  • Bold (Ctrl+B) - to make text bold
  • Italic (Ctrl+I) - to italicize text
  • Underline (Ctrl+U) - to underline text
  • Font Name - to select the font for the subject
  • Size - to specify the font size
  • Paragraph Style - to select a paragraph style
  • Zoom - to zoom in and zoom out the view
  • Foreground Color - to set text color
  • Background Color - to set the background color
  • Print - to print the text

At the bottom of the window, you will find the following buttons:

  • Update - You can click this button to save the date specified.
  • Close - You can click this button to close this window.
