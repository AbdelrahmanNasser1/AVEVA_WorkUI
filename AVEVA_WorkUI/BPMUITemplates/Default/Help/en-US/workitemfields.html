Adding Work Item Fields

This window is displayed when you select Work Item Fields from the Settings menu of the Process Designer. It lists all the work item fields that have been added. The application data available in contents or variables can be displayed in the Work items list using work item fields. You can map the contents and variables with the work item fields in this window.

The 'Work Item Fields' property of the Human activities can be used to specify the work item fields to be used for a particular activity.

This field will automatically be displayed in the Work items list once the activity gets executed and the task is assigned to a resource. When the activity is executed, the values of the custom fields are persisted to the database (SKWork Item Fields table).

The Work Item Field window has the following buttons:

  • Add New - You can click this button to add new work item fields. This will open the second window in the Work Item Field user interface where you can add the new work item fields.
  • Close - Click this button to close the window.

Add/Set Work Item Field window

The Add/Set Work Item Field window is used for adding work item fields. This interface is displayed after the actor clicks Add New button in the Work Item Field window.

For each work item field, the following parameters can be set:

  • Name - You can enter the name of the new work item field in this text box.
  • Display Name- You can specify a display name for the new work item field in this text box. The field will be displayed to the actor with this label name in the Work items list.
  • Type - You can specify the data type corresponding to this work item field by selecting an option from this drop down. The options available are "datetime", "decimal", "number" and "string". For datetime option, the supported format is DD/MM/YYYY.
  • Default Value Type - You can specify the type of value corresponding to the new work item field by selecting an option from this drop down. The options available are "SetValue", "Variable", "Content", "Xml Variables" and "Value".
  • Default Value - You can specify the value for the new work item field.
  • Application Level - You can select this check box to configure the work item fields to be displayed at Application Level.
  • Work Tasks Pro - You can select this check box to configure Work Task Pro grouping and view the work items in groups. The Work Tasks Pro check box is enabled only when Application Level check box is enabled.

    Work Tasks Pro scope can be enabled for only one work item field.

    Note: The value of the Work Item Field is displayed based on the AVEVA Work Tasks server language setting.

This window has the following buttons:

  • Add - You can click this to add the new work item field.
  • Cancel - Click this to cancel the settings for the new work item field and close the window.

When you click Add, the Work Item Field window opens with the new field added to it.
