Using 'Template Path' Property

Scenario Solution:

  1. Design a workflow named 'LeaveWF' as shown in the figure below.

  2. Right-click the Invoke Form activity and then select Activity Properties.
  3. Set the properties for Actors group as shown in the following figure.

    Note: Only the required properties are explained here.

  4. Assign Actor(s): Set this property to send notification mail to the manager as shown in the following figure.

  5. Set the properties for Form Information group as shown in the following figure.

    Note: Only the required properties are explained here.

    1. Form Name: Select 'Leave Form' and its version from Form Name property window.
    2. Input Instance XML: Select 'SFFormData' from Input Instance XML drop-down list.
    3. Output Instance XML: Select 'SFFormData' from Output Instance XML drop-down list.
    4. Save as Draft?: Select Yes from Save as Draft? drop-down list. By setting this property as Yes, the Save As Draft option at the bottom right corner in the Form will be provided. This is useful for filling the form and saving it as draft till final submission of the form.
    5. Close the form on click of Save As Draft?: Select Yes from Close the form on click of Save As Draft? drop-down list. By setting this property as Yes, the form will close on clicking the Save As Draft option.
    6. Set the properties for Notification Contents group as shown in the following figure.

    Note: Only required properties are explained here.

    1. Subject: Select 'Leave Form' from Subject property window.
    2. Body: Specified content will be used in the <%#Body%> placeholder in the template file.
    3. Template Path: Select the 'InvokeFormEmail.html' from the path
      [AVEVA Work Tasks Installed Path]\AVEVA\Work Tasks\WorkflowElements\Default\en-US\Templates\FormEmailTemplate\InvokeFormEmail.html and type the same path in the Template Path property.
      Example1: C:Program Files\AVEVA\Work Tasks\WorkflowElements\Default\en-US\Templates\FormEmailTemplate\InvokeFormEmail.html
      Example2:<%#AVEVA Work Tasks InstallPath%>\WorkflowElements\Default\en-US\Templates\FormEmailTemplate\InvokeFormEmail.html
    4. Send Notification Email: Select Yes - HTML option from the drop-down list.

    Note: The user will be able to send an email only if this property is set as Yes - HTML.

  6. After setting these properties, click Save of the Activity Properties window.
  7. Publish the 'LeaveWF' workflow.
  8. Go to Enterprise Console -> Fill Form. The Fill Form window is displayed as shown in the following figure.

  9. Click the 'LeaveForm'. The 'LeaveForm' form is displayed. Fill this form and click Finish.

  10. On filling the form, the mail will be sent to the manager for approval.

  11. In the mail, just click 'Click here' link to open the form in a browser as shown in the following figure.

    Conclusion: Here, the manager will get the direct link to the leave form which is applied by the employee and here manager can approve or reject leave application instead of logging in to the Enterprise Console.
