AVEVA Work Tasks Connector for Information Server

Introduction

Using the AVEVA Work Tasks Connector for Information Server, you can configure the AVEVA Work Tasks Repository. In this document, you will learn about the following:

  • Provide connectivity from AVEVA Work Tasks Connector for Information Server to the AVEVA Work Tasks Repository.
  • Enable AVEVA Work Tasks Connector for Information Server users to access the AVEVA Work Tasks Repository functionality from within AVEVA Work Tasks Connector for Information Server.
  • Enable access to Central Configuration where users can create or edit AVEVA Work Tasks repositories, and use its functionalities.

Configuring AVEVA Work Tasks Repository and AVEVA Work Tasks Farm for AVEVA Work Tasks Connector for Information Server

Configuring Database Credentials for AVEVA Work Tasks

  1. Run the AVEVA Work Tasks Connector for Information Server application (Programs > AVEVA > Work Tasks > Information Server > AVEVA Work Tasks Connector for Information Server).

    The Database Server Name and Database Name appears.

    • Database Server Name: Displays the Information Server Database Server name. This is auto-populated from the Registry and is Read-only.
    • Database Name: Displays the Information Server Database name. This is Read-only.
  2. Select the Windows radio button to test the connection using Windows authentication mode. You need not type User Id or Password if you select this option.

    Alternatively, select the SQL Server radio button to test the connection using SQL Server authentication mode. You need to type User Id or Password if you select this option.

  3. Enter the SQL Server username in User Id.
  4. Enter the SQL Server password in Password.

    Note: The credentials should have row insertion and deletion permission for the SuiteVoyager database.

  5. Click Test Connection.
    • If the connection succeeds, the next button will be enabled.
    • If the connection fails, you need to re-enter the details.

Configuring AVEVA Work Tasks Server to add the Repository Details of AVEVA Work Tasks

  1. To configure the AVEVA Work Tasks server to add the repository details of AVEVA Work Tasks:
    1. Enter the Unique Environment Name.

      The Unique Environment Name, by default, is a combination of the machine name and Information Server, and is editable the first time. Once repository configuration is done, it cannot be edited.

    2. Enter the AVEVA Work Tasks Workflow server address.
    3. Enter the AVEVA Work Tasks user domain.
    4. Enter the user Id of the AVEVA Work Tasks user.
    5. Enter the AVEVA Work Tasks user password.

      Note: Your User Id should be added to the Administrators, SkeltaAdmins, or SkeltaRemoteUsers role for authentication to succeed.

  2. To check if authentication is successful, and to see the list of repositories available, click Test Connection.

    If the authentication is successful, the Repository Name field is populated with repositories available. If the connection fails, you need to re-enter the details.

  3. Select one repository from the drop down. The provisioning screen appears.
  4. Select the provider from the drop-down list, username and password, depending on provider selection. Click Authorize to enable provisioning.

    Using the Advanced Tabs

    The Advanced tab is available for Outbound Connection Info settings.

    Outbound Connection Info: Advanced Options

    Outbound Connector Port: This field can be edited after you provision the AVEVA Work Tasks Repository for the Information Server.

    Outbound Connector Binding:

    TCP: Default. Binding protocol for the connection, preferably used when AVEVA Work Tasks Server is within the same network.

    HTTPS: Secure connectivity over the Internet.

    Operation Timeout: Time period after which the request made to the AVEVA Work Tasks server will end if not connected.

    Connection Life Time: Time period that the connection will last between the Information Server and the AVEVA Work Tasks Server.

    User Provided Name: Repository user provider. This can be Active Directory or AVEVA Work Tasks List.

    Theme: Repository theme, Grey or default.

    Enterprise Site Console URL: The URL used to access the AVEVA Work Tasks Repository from the AVEVA Work Tasks Server.

    Repository Auth User Id: Captured during provisioning and displayed.

    Repository Auth User Password: Captured during provisioning.

    The AVEVA Work Tasks Server and Repository Details screen appears.

  5. Click Connect to configure the repository for Information Server.

    The Manage Connected Workflow Repositories screen appears.

    Manage Connected Workflow Repositories

    - You can add or delete AVEVA Work Tasks repositories using the Add New or Delete buttons respectively.

    - When you connect, this repository is listed in the Information Server under the AVEVA Work Tasks Repositories node. Also, the Central Configuration link of the Repository is created under the AVEVA Work Tasks Farm node.
