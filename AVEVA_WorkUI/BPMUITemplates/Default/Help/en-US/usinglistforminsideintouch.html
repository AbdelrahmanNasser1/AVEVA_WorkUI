Using List Forms Inside InTouch

Follow Step 1 to Step 7 listed in the Initial Steps and continue:

  1. Select ListFormControl in the Galaxy Browser.

  2. Animations. This will open up a new configuration page from where the List Form control can be configured. After configuring the List Form, click "OK".

    You can set the form instance XML for a List Form in Add Item mode dynamically. The InputXML property of the control should be data-binded to the UDA containing the form instance XML. For more information about the procedure, refer to Passing Contextual Information in Forms from InTouch.

  3. Save and close the Editor.

  4. Select the instance and deploy in the ArchestrA IDE.
  5. Double-click the $InTouchViewApp instance (aaReactorDemo) under ViewEngine from the ArchestrA IDE.

    The InTouch WindowMaker appears.

  6. Create a new window for the Lists control.

  7. Enter the name as "ListForm" and click OK.

  8. Right-click inside the newly opened window and select "Embed ArchestrA Graphic" option.

    This opens up the ArchestrA Galaxy Browser.

  9. Select the Instances tab and select the object to which the graphic symbol has been added.
  10. Select the name of the graphic object (ListForm) that has to be displayed.

  11. Enable security for the Lists window if required.

  12. Save the window and exit InTouch WindowMaker.
  13. Save and close the object in the ArchestrA IDE.
  14. Deploy the $InTouchViewApp instance.
  15. Open the InTouch Viewer and navigate to Special-> Security->Log On.
  16. Enter the credentials and click OK.

  17. Open the created List Form window from the File menu.

    The Lists Control loads in the window.
