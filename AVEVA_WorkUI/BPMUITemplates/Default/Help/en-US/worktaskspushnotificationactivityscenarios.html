Work Tasks Pro Push Notification Activity Scenarios

Activity in waiting state

The activity will be in waiting state, when the first attempt to send notification fails and the property Retry on Failure is set to Yes.

When the first attempt by the activity to send the notification fails, an entry is made into the database. This entry is read by the ArchestrA Mobile Notification Service which in turn attempts to re-send the notification based on the value entered in the Retry Count property.

The activity will be completed only after all the attempts to retry the notifications are made.

Activity output is successful when Recipient Resource is inactive/blocked/not logged in

When multiple recipient resources are configured in the Work Tasks Pro Push Notification Activity and one or more recipient is inactive/blocked/not logged in, the activity will be considered as successful if the notification is sent to all other active recipients.

Unsuccessful activity output scenarios:

  • First attempt to send notification fails and Retry on Failure is set to No.
  • When Retry on Failure is set to Yes and Activity is successful is set to If notification is sent to at least one device: If notification is not sent to any device even after consecutive attempts to resend notification based on the value specified in Retry Count.
  • When Retry on Failure is set to Yes and Activity is successful is set to If notification is sent to all devices: If notification is not sent to one or more devices even after consecutive attempts to resend the notification based on the value specified in Retry Count.
