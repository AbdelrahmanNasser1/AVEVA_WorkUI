Creating a New Folder Using Xml Variable

If your list is version enabled, you can save the item into a folder using System Internal variables. This can be done at the time of setting the value for Update Variable activity.

Assume that you need to create a folder in Workflows list which is version enabled.

To create a new folder using XML Variable

  1. Right-click the Start activity and select Activity Properties from the options.
  2. Click the edit icon in the XMLVariables field. The following screen is displayed.

  3. For the version enabled list, you can see the list of schema in a tree-view under System:Internal. Nodes include Save As NextVersionAsDraft, NextVersionAsPublished, AsFolder and so on.
  4. Drag and drop Update Variable activity. Click the edit icon in the Set Variables field.
  5. Click the Select link and then click the drop-down that appears.
  6. Assume that you want to save the item Description as a folder with the folder name as 'AVEVA Work Tasks'. So, select Description from the tree-view and enter the value as 'AVEVA Work Tasks' in the right-hand side text box.
  7. To save this item as folder, click the drop-down.
  8. Select AsFolder from the SaveAs options under System:Internal.
  9. Then enter the value as 'True'

  10. Click OK. This will update the item Description and also will create a folder with the name 'Manager'.

    Note: Folder names should not have special characters like such as, :, ", \, \, " ", ', <, >, ;, |, *, /, {, }, and ?.
