Auto Mapping of Active Directory Providers

If more than one Active Directory provider is mapped to one Repository then by default, mapping will happen between the active directory providers based on the Display Name property. If user does not want the mapping, then uncheck the Auto Mapping flag from Repository Provider Search page for the User Entity.

By default, the entry in the Properties column of the SKRepository table

<property key="AutoMapActiveDirectoryProviders"><![CDATA[True]]></property>

This indicates auto mapping will happen across the active directory providers once SharePoint Web Application is connected to a repository.

Steps for UnMapping

If user does not want Auto Mapping between providers, do the following:

  1. Log on to the Central Configuration.
  2. Click Repository-Provider List. The Repository-Provider List page is displayed with the list of repositories name.
  3. Expand the required repository where you do not want AutoMapping between Active Directory providers.
  4. Expand UserEntity.
  5. Click the DefinitionXML link to open the DefinitionXML file. Deselect the Do you want to map active directory providers check box at first time, and do the respective operations.
  6. Go to the Farm Database. Now the entry in the Properties column of the SKRepository table is

    <property key="AutoMapActiveDirectoryProviders"><![CDATA[False]]></property>


Steps for Auto Mapping

  1. Log on to the Central Configuration.
  2. Click Repository-Provider List. The Repository-Provider List page is displayed with the list of repositories name.
  3. Expand the required repository and UserEntity where you want AutoMapping between Active Directory providers.
  4. Click the DefinitionXML link to open the DefinitionXML file. Select the Do you want to map active directory providers check box.
  5. Check the Edit the definition xml. The Update button will be enabled.
  6. Click Update.
  7. Select the respective provider, and then click DefinitionXML link to open the DefinitionXML file.
  8. Check the Edit the definition xml. The Update button will be enabled.
  9. Clicking Update will map the across the active directory providers.


Notes:

- If user does not want Auto Mapping between providers, deselect the "Do you want to map active directory providers?" check box for first time and do the respective operations similar to before updating provider DefinitionXML.

- Once it is mapped, unchecking the option will not remove the existing mapping.

- Before connecting Microsoft SharePoint Web Application to a repository, make sure to deselect the check box if you do not want AutoMapping between Active Directory providers.

- If the user does not want to AutoMapping between Microsoft SharePoint and Active Directory provider, then go to Farm Database, open SKRepository table, update the following property to False in the Properties column.
<property key="automapMicrosoft SharePointandactivedirectoryproviders"><![CDATA[False]]></property>
