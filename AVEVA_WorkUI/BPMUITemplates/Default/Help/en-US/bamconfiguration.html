BAM Configuration

The Business Activity Monitoring Configuration enables business analysts to setup critical performance measurement criteria for a workflow.

Performance criteria can be setup to provide the following information:

  • Overdue Analysis
  • Performance Analysis

The BAM Configuration window has the following five tabs:

  • Overdue Analysis: Use this tab to setup thresholds for total pending workflows and pending in different time slots.
  • KPI Data Points: Use this tab to set up KPI data points such as activities pending, workflow status, and other measurement points.
  • KPI(s): Use this tab to set up threshold limits for each data point.
  • Performance: Use this tab to specify work types that must be measured for performance analysis.
  • Alert Configuration: Use this tab to create rules for both Overdue and KPI based on the threshold value set, set the time for the alert to be triggered, and set an action, if the rule is satisfied.
  • Basic Configuration: Use this tab to schedule the analysis. You can also select Process Now for immediate reporting.

Steps to Configure BAM

  1. Define the schedules at which the analysis on workflow execution must be performed.
  2. Configure Overdue Analysis:
    1. Define threshold limits for all pending workflows for Overdue Analysis.
    2. Define data points for Key Performance Indicators to measure.
    3. Define threshold levels for each KPI at different time slots.
  3. Configure Performance Analysis:
    1. Define type of work to be measured, the type of work to be analyzed .
    2. Identify work item fields to compute averages for comparative analysis .
  4. Use the BAM Dashboard to view the Overdue Analysis.
  5. Use the BAM dashboard to analyze Performance. You can also set benchmarks for a resource or an activity to measure resource performance against the benchmark.
