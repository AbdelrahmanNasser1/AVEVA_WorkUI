Work Tasks Pro Push Notification Activity

Activity Description

The Work Tasks Pro Push Notification activity is used to send notifications to the Work Tasks Pro application. When this activity is added and configured in a workflow along with other activities which need human interaction, a notification is sent to the actor intimating the actor that action is required on the assigned task.

Activity Properties

Name and Description

The properties in the Name & Description are used to specify the name and description for the activity.

Name: This property is used to specify a name for the activity.

Property Type: Optional (This property needs to be set only if necessary.)

Description: This property is used to specify a brief description to be displayed for the activity.

Property Type: Optional (This property needs to be set only if necessary.)

Configuration

The properties in the Configuration group allows you to configure the functionality of the activity.

Recipient Resource: This property is used to configure the resources who have to receive the notification. A maximum of 16 resources can be configured as recipients to receive notification. If more than 16 resources are configured, then the 16 active resources will receive the notifications.

You can also build an expression to configure the recipient, this expression will be evaluated at runtime and the recipient will be configured.

Property Type: Mandatory (This property needs be set to add the recipients.)

For more information on Recipient Resource, see Configuring Recipient Resource.

Notification Type: This property allows you to configure custom notification type based on the device and operating system used.

Property Type: Mandatory (This property needs to be set to configure the type of notification specific to a device.)

For more information on Notification Type Setting, see Setting Notification Type.

Message: This property allows you to configure the content of the notification message.

You can also build an expression to configure the notification message. The message will be updated dynamically at runtime.

Property Type: Mandatory (This property must be set to configure the content of the notification.)

Retry on Failure?: When this property is set to Yes and the notification is not sent, attempts to re-send the notification will be made based on the value defined in the Retry Count property.

By default it is set to Yes.

Property Type: Optional (This property needs to be set only if necessary.)

Retry Count: Number of re-tries that needs to be attempted to send the notification. The value specified for this property should be from 1 to 10 only.

Property Type: Optional (This property needs to be set only if necessary.)

Activity output is successful: This property lets you decide when the activity is successful. The available options are If notification is sent to at least one device and If notification is sent to all devices.

  • When If notification is sent to at least one device is selected, the activity will be successful only when the notification is sent to at least one device.
  • When If notification is sent to all devices is selected, the activity will be successful only when the notification is sent to all the devices

Exception Behavior

Each activity has the Raise Error on Unhandled Linked Output property. Selecting one of the property shows all the default mapped error outputs that are configured in red with the respective check boxes selected.

Looping Behavior

The property in the Looping Behavior group has been described below. You can use this property to specify the looping behavior of the activity.

Loop through this action if output is - This property is used to specify the condition for looping the activity by selecting an activity output from the drop down.

Property Type: Optional (This property needs to be set only if necessary.)
