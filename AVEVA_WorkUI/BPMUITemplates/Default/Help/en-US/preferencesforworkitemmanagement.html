Preferences for Work Item Management

When the volume of work items is high, it is essential to minimize the processing time as much as possible. AVEVA Work Tasks provides properties at the control level to set preferences for work item management. These properties enable you to expedite processing by defining how the work items can be managed and acted upon.

The two properties for work item management are as follows:

  • Auto-ownership on Row Click: This property enables you to allow users to take ownership of a work item by clicking on the item in the Work items list. If this property is set to true, the option Take Ownership in the Manage Work Items field in Work Item Details will not be available.
  • Enabling Multi-Select Option in Work Item List: This property provides users with an option to select multiple work items for processing in a batch. For example, selecting multiple work items for approval.

    Note: The default view of the Work items list in the Enterprise Console cannot be customized. The above customization applies only to the work items lists are created by embedding the Work Item List Control in an ASPX page. For more information about using AVEVA Work Tasks Controls as a part of a project, see Using AVEVA Work Tasks Controls in Applications in the AVEVA Work Tasks Developer Guide.
