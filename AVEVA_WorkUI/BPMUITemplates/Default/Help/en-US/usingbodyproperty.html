Using 'Body' Property

Scenario Solution:

  1. Design a workflow named 'LeaveWF'.

  2. Right-click Invoke Form activity and then select Activity Properties.
  3. Set the properties for Actors group as shown in the figure below.
    Here only required properties are explained.

    • Assign Actor(s): Set this property as shown in the following figure.

  4. Set the properties for Form Information group as shown in the following figure.

    Note: Here only required properties are explained.

    • Form Name : Select 'Leave Form' and its version from Form Name property window.
    • Input Instance XML: Select 'SFFormData' from Input Instance XML drop-down list.
    • Output Instance XML: Select 'SFFormData' from Output Instance XML drop-down list.
    • Save as Draft?: Select Yes from Save as Draft? drop-down list. By setting this property as Yes, the Save As Draft option at the bottom right corner in the Form will be seen. This is useful for filling the form and saving it as draft till final submission of the form.
    • Close the form on click of Save As Draft?: Select Yes from Close the form on click of Save As Draft? drop-down list. By setting this property as Yes, the Form will close on clicking the Save As Draft option.
  5. Set the properties for Notification Contents group as shown in the following figure.

    Note: Here only required properties are explained.

    • Subject : Specify the appropriate subject from Subject property window.
    • Body: Click the edit button in the Body property to open the Body property window.
    1. Compose the body text.
    2. Click Enterprise Console Explorer(plug-in) icon to select the URL. The following window is opened with list of Enterprise Console names. Select the Local URL and then click Select. The selected URL will be appended in the Body window wherever the cursor is pointed as <%EnterpriseConsole,LocalURL%>.

      The composed mail will be as shown in the figure below.

      Click Update of the Body window property to save the contents.

    • Send Notification Email: Select Yes - HTML option from the drop-down list.

      Note: You will be able to send an email only if, this property is set as Yes - HTML or Yes. If it is Yes, then the mail will be sent as a plain text. If it is Yes- HTML, then the mail will be sent as HTML format.

  6. After setting these properties, click Save of the Activity Properties window.
  7. Publish the 'LeaveWF' workflow.
  8. Go to Enterprise Console > Fill Form. The Fill Form window is displayed as shown in the following figure.

  9. Click on the 'LeaveForm'. The 'LeaveForm' form is displayed. Fill this form and then click Finish.

  10. On filling the form, the mail will be sent to the manager for approval.

    <%EnterpriseConsole,LocalURL%> text added in the Body tag is replaced with Click here hyperlink text.

    Click here text can be changed from the [AVEVA Work Tasks Installed Path]\Locales\en-US\EmailText.txt to the required text.

  11. In the mail, just click on 'Click here' link to open the form in a browser as shown in the following figure.

    Conclusion: Here manager will get the direct link to the leave form which is applied by the employee and here manager can approve or reject leave application instead of logging in to the Enterprise Console.

    Note: You can act on a task only first time. Next time, it will display the following message:

"Duplicate Response

Your response for the task <Activity Name> of <Date> application's <Workflow Name> workflow was recorded earlier! Workflow engine did not consider this duplicate response. "
