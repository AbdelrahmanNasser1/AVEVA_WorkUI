Invoke Form - Individual actor's response collaboration in Output Instance XML

You can use this property to collaborate with an individual actor's responses in the Invoke Form activity. You can configure this property only when the activity has multiple actors and you want to store the individual actor's responses as a collection in an Output Instance XML. When you use this property, the entire OutputXML is not updated. Only the nodes specified in the Collaboration/Individual Response Node is copied to the All Actors Responses Node of the Output instance XML.

The following image shows the input fields of the collaboration mode property:

If the Invoke Form Activity is not configured in collaboration mode, and if the task is assigned to multiple users, then only the submission changes made by the last user will be considered.

Inputs for Collaboration/Individual Response Node and All Actors Responses Node must be either a base form or a panel form with XML node. You need to set Collaboration/Individual Response Node, which is the node used to capture the response of individual user. Also All Actors Responses Node is the node in the output instance XML to which the individual response node will be appended.

Steps to capture the Individual Actor's response collaboration in Output Instance XML

  1. Click the edit button corresponding to Individual actor's response collaboration in Output Instance XML.

    The Invoke Form-AdvancedResponseStorage dialog box appears.

  2. In the Collaboration/Individual Response Node, select the node in the form which will be used to capture the response of the user.
  3. You can edit only the controls of the selected node. All the other controls will be read-only.
  4. In the All Actors Responses Node, select the node in the output instance XML to which the above selected response node is to be appended.
  5. Click OK.

Note: You can even include a User control to capture the User ID by default. This can be either visible or invisible to the end-user. To create a user control in the Form, you will have to populate the logged in user value in a form to a hidden control, so that in a workflow, a decision action can be performed based on the user who has filled the form.

Example scenario:

The Telematics Company allows its employees to post articles. The articles posted can be viewed by a group of users defined in the workflow and these users can leave their comments on the article. These article viewers can also view the comments provided by others.

To achieve this, the user has to go through six steps. For each step, the respective XML file name with code is given below, which you can use.

Note:

1. In this scenario, we have used the repository with AVEVA Work Tasks List as the user provider.

2. Create XML files with the same filename as given below. Place all the files in a single folder for ease of use.

  1. Create a list of users in the User's list with the group name 'ArticleSubscribers'.
    1. In the Enterprise Console, click the Menu button, click Settings, and then click Users. The Users page appears.
    2. Create some users with ArticleSubscribers as their role.
  2. Create a List to store the submitted articles.
    1. In the Enterprise Console, click the Menu button, click Tools, and then click Manage List. The Manage List page appears.
    2. Create a new list named Telematics Article to store the submitted articles.
    3. Click on Save & Continue to go to the Select Table page.
    4. Select the Generic Table option.
    5. Click Next to go to the List Designer.
    6. Import the list definition from Telematics Article List.xml using Import menu option in the List Designer.
    7. Click Submit, to generate a default form for the list.
    8. Click Yes, to edit the default form which is generated. To change the look and feel of this default form, import the Telematics Article_ListForm.xml from Import menu option in forms designer. Now your form looks like the image given below in the forms designer.
    9. Submit and publish the form.

  3. Create a Form to show the submitted article to the 'ArticleSubscribers' group, using which the users will be able to add a comment, as well as view comments given by other users.
    1. In the Enterprise Console, click the Menu button, click Tools, and then click Manage Forms. The Manage Forms page appears.
    2. Create a form named TelematicsArticlesReview.
    3. Import the form definition from TelematicsArticlesReview_Form.xml using the Import menu option in the Forms designer. If you notice, your form is again an exact copy of the Telematics Article form, but with an extra panel of 'Leave a Comment' section.
    4.  You can see a tab in this form, where the first tab is to leave the comment by the Articlesubscriber group, and the second is to view the comments posted by other s in the group. If you do not need to show the others comment section, you can remove the views other's comment  section.
    5. Check the  mapped XML nodes (of parent and child controls) of both the  'Leave a comment' and 'Review Comments' control. Here you might have noticed that the child nodes have the same mapped XML nodes where as only for the parent it differs.
    6. To append/show a row by default for 'Leave a comment' panel, you can notice that a script is defined for the category drop down. Notice that this can also be achieved without scripting by setting the 'Rows to be added by default' as '1' and 'Allow default rows to render on XML Load?' as 'Yes' in the Advanced tab of 'Leave a comment' panel properties.
    7. Submit and Publish the form.
  4. Create an Initial Form to submit the article.
    1. In the Enterprise Console, click the Menu button, click Tools, and then click Manage Forms. The Manage Forms page appears.
    2. Create a Workflow Initiator Form named Telematics Article Submission.
    3. Import the form definition from Telematics Article Submission Form.xml using the Import menu option in the Forms designer. You will notice that your form is an exact copy of the Telematics Article form, but without the Review comments section.
    4. Submit and Publish the form.
  5. Associate an Article Submission Form to the Article Review Workflow.
    1. On submission, a page related to Form Binding opens, where the user can select a task as designing a new workflow or selecting an existing one.
    2. Select Design a New Workflow and provide the workflow name as ArticleReview.
    3. Select Design this Workflow now in the next page.
    4. From the Process Designer, use the Import option from Tools menu and import the workflow from ArticleReview Workflow.xml.
    5. Publish the workflow.
  6. Execute the form to see how each individual actor's responses collaborate.
    1. In the Enterprise Console, click the Menu button, and then click Fill Form. The Fill Form page appears.
    2. Select the Telematics Article Submission form, fill the details, and submit the form.
    3. Go to the Inbox of each ArticleSubscribers, leave a comment, and submit the form.
    4. You can view all user's comments under 'View Other's comments' section.
