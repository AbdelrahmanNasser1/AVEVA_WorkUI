Using Forms Inside InTouch

Follow Step 1 to Step 7 listed in the Initial Steps and continue:

  1. Select the control in the editor and double-click Forms Configuration under Animations.

    This will open up a new configuration page from where the Forms control can be configured.

  2. Under Fill Form Settings, perform the following:
    1. In the Form Name field, enter the form name.
    2. In the Form Version field, enter the version of the form, which must be considered by the Fill Form.

      For the Fill Form control to consider the latest published version of the form, enter the value as 0 or leave this field blank.

      If you enter an invalid version, then the "The given version does not exist for the form" message appears.

  3. After configuring the Forms, click OK.
  4. Save and close the Editor.

  5. Select the instance and deploy in the ArchestrA IDE.
  6. Double-click the $InTouchViewApp instance (aaReactorDemo) under ViewEngine from the ArchestrA IDE.

    The InTouch WindowMaker appears.

  7. Create a new window for the Forms control.

  8. Provide the name and size for the window.

  9. Right-click inside the newly opened window and select the Embed ArchestrA Graphic option.

    This opens up the ArchestrA Galaxy Browser.

  10. Select the Instances tab and select the object to which the graphic symbol has been added.
  11. Select the name of the graphic object (Forms) that has to be displayed.

  12. Enable security for the Forms window, if required.

  13. Save the window and exit InTouch WindowMaker.
  14. Save and close the object in the ArchestrA IDE.
  15. Deploy the $InTouchViewApp instance.
  16. Open the InTouch Viewer and navigate to Special-> Security->Log On.
  17. Enter the credentials and click OK.

  18. Open the created Forms window from the File menu.

    The Forms Control loads in the window.


    Note: InTouch does not support the execution of window.resizeTo(x, y) JavaScript function.
