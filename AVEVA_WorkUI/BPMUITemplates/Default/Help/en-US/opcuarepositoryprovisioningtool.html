OPC UA Repository Provisioning Tool

You can use the AVEVA Work Tasks OPC UA Repository Provisioning Tool to provision a workflow repository for OPC UA. Only the provisioned repositories can be connected to OPC UA. Workflow repository provisioning for OPC UA should be done only after installing AVEVA Work Tasks.

The AVEVA Work Tasks OPC UA Repository Provisioning Tool is installed during the installation of AVEVA Work Tasks Extension for OPC UA.

To Provision a Repository for OPC UA:

  1. Run the AVEVA Work Tasks OPC UA Repository Provisioning Tool from the Start menu > All Programs > AVEVA > Work Tasks folder.

    The AVEVA Work Tasks Extension for OPC UA Configuration window appears. This window displays all the repositories from the system.

  2. Right-click on any cell column of the particular repository, and then select the Provision option.

    Alternatively, you can click the row arrow corresponding to the particular repository. When the repository row gets selected, click Provision.

    The following message appears:

    Provisioning will modify the repository database by adding new tables and adding or updating of records in the existing tables. It is highly recommended to take a backup of the repository database before provisioning, as there is no option to revert the provisioned changes. To revert back the repository database to its previous state, use the Database Restore option. Do you want to Continue?

  3. Select Yes to provision the workflow repository.

    The repository provision is successful.

  4. Click Close.

To UnProvision a Provisioned Repository:

To unprovision an already provisioned repository, right-click on any cell column of the particular repository, and then select the Unprovision option.
